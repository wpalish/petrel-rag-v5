# tests/ — Тестовый набор для RAGAS-оценки

## Структура `testset.json`

30 вопросов на русском и казахском языках, сгруппированных по типам:

| Тип | Кол-во | Назначение |
|-----|--------|------------|
| Базовые факт-вопросы (RU) | 10 | Проверка поиска конкретных фактов |
| Базовые факт-вопросы (KZ) | 5 | Проверка казахского языка + гибридного поиска |
| Процедурные (RU/KZ) | 7 | Многошаговые процедуры — проверка PDR |
| Edge-cases "нет ответа" | 4 | Должны вернуть no-answer |
| Неоднозначные | 4 | Должны попросить уточнить или дать осторожный ответ |

## Поля вопроса

```json
{
  "id": "q01",
  "question": "Сколько дней длится отпуск?",
  "language": "ru",
  "expected_no_answer": false,
  "expected_doc_id": "положение_об_отпусках_...",   // опц.
  "expected_keywords": ["24", "календарных"],        // опц.
  "notes": "Базовый вопрос"
}
```

## Запуск оценки

### Через REST API

```bash
# Все 30 вопросов
curl -X POST http://localhost:8000/evaluation/run

# Только первые 5 (быстрый smoke-test)
curl -X POST "http://localhost:8000/evaluation/run?limit=5"
```

### Через скрипт

```bash
./scripts/run_evaluation.sh
```

## Метрики

| Метрика | Что измеряет | Цель |
|---------|--------------|------|
| `avg_faithfulness` (RAGAS) | Насколько ответ подтверждается контекстом | ≥ 0.85 |
| `avg_answer_relevancy` (RAGAS) | Насколько ответ релевантен вопросу | ≥ 0.80 |
| `avg_language_purity` | Доля ответов без смешения языков | ≥ 0.95 |
| `avg_no_answer_accuracy` | Корректность "нет информации" на edge-cases | 1.0 |
| `avg_citation_accuracy` | Доля ответов с корректными [N] ссылками | ≥ 0.90 |
| `avg_keywords_match` | Доля ожидаемых ключевых слов в ответе | ≥ 0.70 |
| `avg_latency_sec` | Среднее время ответа | ≤ 5.0 |

## Калибровка

После первого прогона:
1. Если `avg_faithfulness < 0.85` → повысить `MIN_RERANK_SCORE` с 0.30 до 0.40-0.50.
2. Если `avg_language_purity < 0.95` → усилить few-shot для казахского в `prompts.py`.
3. Если `avg_no_answer_accuracy < 1.0` → понизить `MIN_RERANK_SCORE` (модель отвечает на edge-cases, хотя не должна).
4. Если `avg_citation_accuracy < 0.90` → проверить формат источников в `rag_engine.py::generate()`.

## Добавление новых вопросов

1. Открыть `tests/testset.json`.
2. Добавить объект в массив `questions` с уникальным `id` (например, `q31`).
3. Заполнить поля согласно структуре выше.
4. Запустить `POST /evaluation/run` — новый вопрос автоматически включится.

## Эталоны

Эталоны (expected_keywords, expected_no_answer) — **предполагаемые** на основе типичных корпоративных регламентов. Перед использованием на реальных данных заказчика — откалибровать под фактическое содержание документов.

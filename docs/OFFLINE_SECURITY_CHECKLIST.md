# Offline Bootstrap & Security Audit Checklist (v2)

**Цель:** Гарантировать, что система может быть развёрнута в полностью изолированном контуре без внешнего доступа к интернету и без телеметрии.

---

## 1. Offline Bootstrap (предварительная подготовка)

### 1.1 Машина с интернетом (подготовка)

- [ ] Установить `huggingface-cli` и `hf-transfer`:
  ```bash
  pip install huggingface_hub hf-transfer
  ```
- [ ] Выполнить предзагрузку моделей (скрипт сделает это автоматически):
  ```bash
  ./scripts/preload_models.sh
  ```
  Будут скачаны:
  - `Qwen/Qwen2.5-7B-Instruct-AWQ` (~5 GB)
  - `BAAI/bge-m3` (~2.3 GB)
  - `BAAI/bge-reranker-v2-m3` (~2.3 GB)

- [ ] Скопировать папку `./models` на целевой сервер (через USB / внутреннюю сеть / air-gap носитель).

- [ ] (Опционально) Создать локальный Docker registry:
  ```bash
  docker pull qdrant/qdrant:v1.12.4
  docker pull vllm/vllm-openai:v0.6.4.post1
  docker pull ghcr.io/huggingface/text-embeddings-inference:1.6
  docker pull python:3.11-slim
  docker save python:3.11-slim qdrant/qdrant:v1.12.4 vllm/vllm-openai:v0.6.4.post1 ghcr.io/huggingface/text-embeddings-inference:1.6 | gzip > images.tar.gz
  ```
  На целевом сервере: `docker load < images.tar.gz`

### 1.2 На целевом изолированном сервере

- [ ] Разместить `./models` в директории проекта.
- [ ] `cp .env.example .env`
- [ ] Установить `HF_HUB_OFFLINE=1` в `.env` (ТОЛЬКО после первого успешного старта).
- [ ] `docker compose up -d --build`
- [ ] Проверить, что ни один контейнер не пытается выйти в интернет (см. §2).

## 2. Telemetry & Outbound Audit

### 2.1 Обязательные переменные окружения

```bash
HF_HUB_OFFLINE=1                 # Только ПОСЛЕ загрузки моделей
VLLM_NO_USAGE_STATS=1
OLLAMA_NO_TELEMETRY=1
```

### 2.2 Запуск аудита

```bash
./scripts/audit_telemetry.sh
```

Скрипт проверяет:
1. Переменные окружения в backend.
2. Health vLLM и Qdrant.
3. tcpdump в течение 30 секунд — ищет исходящий трафик за пределы docker-сети.
4. Логи backend на предмет подозрительных ключевых слов (huggingface.co, openai.com, telemetry, sentry).

**Ожидаемый результат:**
- Section 4 (tcpdump) — **пусто** (нет исходящего трафика).
- Section 5 (logs) — **нет** подозрительных ключевых слов.
- `HF_HUB_OFFLINE=1` активен.

## 3. Smoke Test на изолированном сервере

1. [ ] Загрузить 2–3 тестовых документа через UI.
2. [ ] Задать 5–7 вопросов (включая вопросы "без ответа").
3. [ ] Проверить:
   - Ответы содержат корректные цитаты (`doc_name`, `page_number`, `section_title`).
   - На вопросы без информации возвращается "В предоставленных документах нет информации...".
   - Нет ошибок загрузки моделей в логах.
4. [ ] Перезапустить все контейнеры и убедиться, что индекс сохранился (volume `qdrant_data`).
5. [ ] Проверить, что snapshot создаётся после каждой индексации.

## 4. Формальные требования к источникам (UI + API)

В ответе `/query` и в UI **обязательно** должны отображаться:

- `doc_name` — имя файла
- `page_number` — номер страницы (для PDF)
- `section_title` — заголовок раздела
- `paragraph_index` — номер абзаца
- `score` — vector similarity
- `rerank_score` — скор реранкера (после normalize, в [0,1])

Пример в Gradio:
```
[1] Положение_о_командировках.pdf (стр. 7, раздел "3.2 Порядок оформления")
   ▸ rerank: 0.847 · vector: 0.792
   ▸ фрагмент: Служебная записка оформляется...
```

## 5. Калибровка параметров

| Параметр | Значение по умолчанию | Статус | Рекомендация |
|----------|----------------------|--------|--------------|
| `MIN_RERANK_SCORE` | 0.30 | **Калибруемый** | Старт с 0.30. После 20–30 тестовых вопросов скорректировать. |
| `TOP_K` | 8 | Калибруемый | Сколько кандидатов из Qdrant |
| `RERANK_TOP_K` | 5 | Калибруемый | Сколько оставить после reranker |
| `CHUNK_SIZE` | 512 | Калибруемый | Размер child-чанка |
| `PARENT_CHUNK_SIZE` | 1400 | Калибруемый | Размер parent-чанка для генерации |
| `LLM_TEMPERATURE` | 0.1 | Калибруемый | Ниже = меньше галлюцинаций |

**Правило:** Все пороги вынесены в `app/config.py` и помечены комментарием `# CALIBRATABLE`.

## 6. Sparse часть bge-m3

В MVP используется только **dense** часть bge-m3.

Для перехода на полноценный hybrid:
- Добавить вычисление sparse вектора в `app/ingestion.py::embed_texts()`.
- Сохранять в `vector["sparse"]` (поле уже зарезервировано в схеме).
- Реализовать RRF (Reciprocal Rank Fusion) в `app/rag_engine.py::retrieve()`.

Это задача этапа оптимизации (после MVP). **Миграция коллекции не требуется.**

Поиск по базе знаний,

Локальная RAG-система: отвечает на вопросы по внутренним регламентам на казахском и русском, со ссылками на первоисточники. Open-Source LLM, гибридный поиск, реранкер — без единого внешнего API.

Код на GitHub → Почему это безопасно

Возможности

Production-архитектура, а не туториал на LangChain.

🔒

Полная локальность

LLM, эмбеддинги, реранкер и вектор-БД работают на сервере компании. Нет OpenAI, Anthropic, Cohere.

📖

Цитаты на первоисточники

Каждое утверждение сопровождается ссылкой [N] на документ, страницу и раздел.

🧠

Гибридный поиск + реранкер

Dense (bge-m3) + BM25 + RRF + bge-reranker-v2-m3. Смысл и точные формулировки одновременно.

🌍

Казахский и русский

Автоопределение языка вопроса, ответ без смешения языков.

🛡️

Анти-галлюцинации

Порог релевантности + строгий промпт + программный «нет информации» при отсутствии ответа.

📊

Оценка и мониторинг

Локальный eval-харнесс (без OpenAI) + Prometheus-метрики и дашборды Grafana.

Как работает

Гибридный конвейер извлечения — это и есть «интеллектуальность».

Вопрос KZ/RU→ Dense (bge-m3)+ BM25→ RRF fusion→ Reranker→ Порог + PDR→ LLM → ответ + [цитаты]

Безопасность — главное требование

Данные и телеметрия не покидают сервер. Это не опция, а архитектурный принцип.
